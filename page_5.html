技術選択の理由

Flutter

単一のコードベースでiOSとAndroidの両プラットフォームに対応可能

豊富なUIコンポーネントと高い描画性能

直感的で美しいユーザーインターフェースの実現が可能

開発効率を最大化し、ユーザーに迅速にアプリを届けることができる

OpenAI API

レシート解析における高精度な項目抽出

ユーザーの多様なニーズに応じた献立提案

強力な自然言語処理能力と画像認識能力

従来のAIでは難しかった複雑なタスクを可能にする

ローカルデータベース

ユーザーの機密性の高い家計情報や食生活データの扱い

セキュリティとプライバシーを最優先に考慮

オフライン環境でもデータにアクセス・管理できる利便性

外部サーバーへのデータ送信を最小限に抑えて情報漏洩のリスクを低減

開発過程での技術的チャレンジ

OpenAI API: プロンプトエンジニアリングの最適化に苦労

Flutter: 複雑なUIアニメーションの実装に挑戦

ローカルDB: ローカルデータベースとクラウドサービス間のデータ同期の設計

これらの技術を学習する過程では、一つ一つ解決していくことで、より堅牢でユーザーフレンドリーなアプリ開発の知見を深めることができました
